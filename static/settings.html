      Hierarchy

      Classification tiers, outermost first. Each tier needs a code property and/or a name property. A tier named "Building" powers the building filter.

      Builtin properties

      Canonical property name → each source's raw property name (e.g. canonical "Area" backed by revit's "Fläche"). Only needed when a raw name differs from the canonical one.

      Room label

      Ordered fields shown on each room in the viewer. $name and $id are the room's own name/id; anything else is a property name.

      Milestones

      Named dates with data snapshots pinned to them. The viewer's milestone dropdown lists these; a pinned model is served from that snapshot instead of its latest, and an unpinned model doesn't appear in the milestone view. Snapshot ids are what's stored on the server.

      dRofus QA fields

      Per-column type/comparison declarations for the validation report. Labels come from row 1 of the dRofus CSV — run "check" above to load them.

      Colour plans

      Named room-colouring configs the viewer's colour picker offers. At most one may be active (the picker's default; "No colour" always overrides). All colouring is computed in the browser — the server just stores these. This pass edits property compare plans (compare two room properties, colour by the result); property names come from the current project's rooms.
